# `eval/` ‚Äî `eval-jss` precision harness

This directory holds the Python package and supporting data files for
`eval-jss`, the companion CLI that measures per-rule precision of
`jss-lint` against a real-world corpus of JSS manuscripts. Spec and
design artifacts live under
[`specs/002-eval-jss-harness/`](../specs/002-eval-jss-harness/).

The harness implements Constitution ¬ßVI (‚â•90% precision per rule) as an
enforceable, empirical gate, and ¬ßXII (reproducible corpus) via a
pinned `corpus-manifest.csv`.

## Runtime state vs. checked-in state

- **Gitignored**: `eval.db`, `eval.db-shm`, `eval.db-wal`, and any
  `examples/**/.sha256` marker files dropped by `corpus fetch`.
- **Checked in**: `corpus-manifest.csv`, `review-skip-list.toml`,
  `report.csv`, and `corpus-manifest-gaps.csv` ‚Äî these are the durable,
  diffable records that make precision claims reproducible.

## Adding a paper to the corpus (MVP procedure)

1. Pick a CRAN package whose vignette uses `\documentclass{jss}` or
   `\usepackage{jss}` (e.g., `lme4`, `MASS`, `limma`, `zoo`).
2. Download the current tarball from CRAN `Archive/`. Extract and copy
   the `.tex` + `.bib` into `examples/cran_<pkg>_<version>/` along
   with a one-paragraph `README.md` recording the URL, version, and
   (if available) JSS DOI from `inst/CITATION`.
3. Add a manifest row pointing at the tarball URL with its SHA256.
4. Run `eval-jss scan && eval-jss corpus status` ‚Äî both should succeed.

The current `examples/` directory contains three placeholder
vignettes that exist solely to exercise the plumbing end-to-end; see
[`examples/README.md`](../examples/README.md) for the handoff policy.

## Running the harness

Full walkthrough: [`specs/002-eval-jss-harness/quickstart.md`](../specs/002-eval-jss-harness/quickstart.md).

Short form:

```sh
eval-jss init                                    # idempotent schema
eval-jss scan --corpus examples/                 # persist violations
eval-jss human-review                            # interactive labelling
eval-jss review --confidence-threshold 0.8       # AI-assisted (Phase B)
eval-jss report --csv eval/report.csv            # print + append history
eval-jss report --by-source                      # per-source breakdown
eval-jss corpus fetch                            # reproduce from manifest
eval-jss corpus status                           # report corpus state
```

Exit codes: `0` all rules at ‚â•90% precision ¬∑ `1` at least one rule
below threshold, or `scan` newly persisted violations, or `corpus fetch`
logged a gap ¬∑ `2` the harness itself could not run.

## The eval-improve loop

One iteration grows the corpus, surfaces rule defects, and shrinks them.
The loop has 9 steps, each available as a custom slash command under
`.claude/commands/eval/` (e.g. `/eval:scan`, `/eval:record`):

1. **Add 50 manuscripts** ‚Äî `/eval:add-corpus` (edits
   `eval/corpus-manifest.csv`, then `eval-jss corpus fetch`).
2. **Scan** ‚Äî `/eval:scan` (`eval-jss scan --corpus examples/ --force`).
3. **AI classifier** ‚Äî `/eval:review`
   (`eval-jss review --confidence-threshold 0.8`).
4. **Human review** (optional) ‚Äî `/eval:human-review`
   (`eval-jss human-review` for the rows the classifier was unsure
   about or the skip-list excluded).
5. **Record** ‚Äî `/eval:record <label>`
   (`eval-jss iterate record <label> [--note ‚Ä¶]`). Snapshots the
   precision report twice ‚Äî full corpus and `--pinned-only` ‚Äî into
   `eval/precision-history.db` and appends a templated section to
   `eval/improvement-log.md`.
6. **Suggest improvements** ‚Äî `/eval:suggest` (edits the iteration's
   "Findings / suggestions" block in `eval/improvement-log.md`).
7. **Plan** ‚Äî `/eval:plan` (fills in the "Plan" block with a checklist
   of concrete todos).
8. **Implement** ‚Äî `/eval:implement` (works through the checklist on a
   branch).
9. **Re-scan + record** ‚Äî `/eval:rescan <label>`
   (`eval-jss scan --force && eval-jss iterate record post-<label>`).
   The new section's "Delta" block diffs against the previous iteration
   so before/after is visible inline.

`--pinned-only` restricts precision stats to violations from the
vignette named in `corpus-manifest.csv::vignette_file` (plus every
sibling `.bib` in the vignette's directory, since `\bibliography{...}`
resolves there). This isolates rule quality on canonical inputs from
noise introduced by other vignettes in the tarball.

`eval/precision-history.db` is append-only and independent of
`eval/eval.db` ‚Äî wiping the raw-violations DB does not erase the
precision trend.
