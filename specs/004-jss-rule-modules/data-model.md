# Phase 1 Data Model — JSS Rule Modules

Every primary type (`Rule`, `RuleCategory`, `ParsedDocument`, `ToolConfig`, `Violation`, `Severity`, `FixSuggestion`) is defined in spec 001 `data-model.md` and **unchanged here** except for a single API-additive amendment: `Severity` grows a third enum value `INFO = "info"` to accommodate the four info-severity rules in spec 003's catalogue.

This spec introduces **three** new in-repo artefacts, all under `src/texlint/journals/jss/`: the generated `_catalogue_data.py`, the shared `rules/_helpers.py`, and the per-category rule modules `rules/<category>.py`.

## 1. `Severity` enum extension

**Location**: `src/texlint/api.py`.

**Change**: add `INFO = "info"` to the existing `Severity` enum.

```python
class Severity(str, Enum):
    ERROR = "error"
    WARNING = "warning"
    INFO = "info"            # NEW — spec 004 Phase 2 amendment
```

**Invariants** (unchanged from spec 001):
- `Severity(...)` construction accepts `"error" | "warning" | "info"`; anything else raises `ValueError`.
- Enum values serialise as their string form (JSON output, eval-harness DB storage).

**Audit** (spec 004 Phase 2 foundational task):
- `src/texlint/output/terminal.py`, `src/texlint/output/html.py`, `src/texlint/output/json_.py` (or equivalent) — confirm INFO renders sensibly (default: treat same as WARNING unless an explicit INFO-tier visual is warranted).
- `eval/db.py` — severity column stores the string; nothing to change.
- `eval/report.py` — confirm INFO precision numbers are reported alongside WARNING/ERROR or explicitly flagged.
- `eval/human_review.py` — confirm INFO violations surface to the reviewer (default: yes, same as WARNING).

## 2. `_catalogue_data.py` — generated rule metadata

**Location**: `src/texlint/journals/jss/_catalogue_data.py`.
**Generated by**: `python -m tools.generate_catalogue_data`.
**Source of truth**: `specs/003-jss-rule-catalogue/catalogue.yaml`.
**Status**: committed artefact; regeneration drift fails CI (`test_catalogue_data_fresh.py`).

### Exports

| Name | Type | Semantics |
|---|---|---|
| `RULES` | `Mapping[str, Mapping[str, object]]` | One entry per **active** rule. Keys are `rule_id`s; values are read-only mappings of per-rule metadata (see sub-shape below). |
| `RETIRED_RULE_IDS` | `frozenset[str]` | Every id in `catalogue.yaml`'s `retired_rule_ids:` field. |
| `ROLLOUT_ORDER` | `tuple[str, ...]` | The 15-category sequence pinned by spec 003 / spec 004 FR-007. |

### Per-rule metadata sub-shape

Every `RULES[rule_id]` value is a `MappingProxyType` with exactly these fields:

| Field | Type | Source in `catalogue.yaml` |
|---|---|---|
| `category` | `str` | `category` |
| `severity` | `Severity` enum | `severity` (mapped from string) |
| `message_template` | `str` | `description` |
| `authority` | `str` | `authority` (one of `jss_cls` / `article_tex` / `style_guide` / `author_instructions`) |
| `authority_ref` | `str` | `authority_ref` |
| `inspects` | `tuple[str, ...]` | `inspects` (list → tuple for immutability) |
| `auto_fixable` | `bool` | `auto_fixable` |

### Generation rules

- File header is `# AUTO-GENERATED from specs/003-jss-rule-catalogue/catalogue.yaml.\n# Do not edit by hand; run \`python -m tools.generate_catalogue_data\`.\n\n`.
- Imports: `__future__ import annotations`, `types.MappingProxyType`, `typing.Mapping`, `texlint.api.Severity`. Nothing else.
- Deterministic output: entries are sorted by `(rollout_order_index, rule_id_counter)` — the same order `render_catalogue.py` uses for markdown. Two runs on the same input produce byte-identical output.
- Severity mapping: `"error"` → `Severity.ERROR`, `"warning"` → `Severity.WARNING`, `"info"` → `Severity.INFO`. Mapping lives in a top-of-generator helper `_SEVERITY_MAP` (not exported from the generated file).
- String escapes: `message_template` values use Python repr with `"` quotes; backslashes double-escaped. `authority_ref` same. No newlines in any string field (catalogue validator rejects multi-line descriptions).

### Invariants

| # | Invariant | Enforcer |
|---|---|---|
| D-1 | Every `rule_id` key in `RULES` matches the regex `^JSS-[A-Z]+-\d{3}$`. | codegen; also `test_catalogue_data_fresh.py` |
| D-2 | `RETIRED_RULE_IDS` is disjoint from `set(RULES)`. | codegen; also `test_catalogue_registration.py` |
| D-3 | Every `category` value in `RULES` appears in `ROLLOUT_ORDER`. | codegen |
| D-4 | `ROLLOUT_ORDER` has length 15 and contains each category from `catalogue.yaml` `categories:` exactly once. | codegen |
| D-5 | The committed `_catalogue_data.py` is byte-identical to what `generate_catalogue_data` would emit given the current `catalogue.yaml`. | `test_catalogue_data_fresh.py` |

## 3. `rules/_helpers.py` — private shared walkers

**Location**: `src/texlint/journals/jss/rules/_helpers.py`.

### Public-to-rules surface (still private to the package — underscore prefix)

| Function | Signature | Purpose |
|---|---|---|
| `_walk` | `_walk(nodelist: Sequence[Node] \| None) -> Iterator[Node]` | Pre-order traversal. Recurses into environment bodies, group contents, math-node contents, macro-argument group contents. Yields every non-None node. |
| `_iter_with_parent` | `_iter_with_parent(nodelist: Sequence[Node] \| None) -> Iterator[tuple[Sequence, int, Node]]` | Yields `(parent_nodelist, index, node)` triples — needed for sibling-context checks. |
| `_macro_args_text` | `_macro_args_text(macro: LatexMacroNode, parent: Sequence[Node], idx: int) -> str` | Extracts the macro's first braced-group arg as plain text. Tries `nodeargd.argnlist` first (pylatexenc-known macros); falls back to the next sibling (unknown macros like `\pkg`, `\proglang`, `\code`). |
| `_next_group_arg` | `_next_group_arg(parent: Sequence[Node], idx: int) -> LatexGroupNode \| None` | Helper for `_macro_args_text` fallback path. |
| `_lineno_col` | `_lineno_col(tex: ParsedTexFile, pos: int) -> tuple[int, int]` | Resolves `(1-based line, 1-based column)` from a source position via `tex.walker.pos_to_lineno_colno`. |
| `_is_inside_verbatim` | `_is_inside_verbatim(ancestors: Sequence[Node]) -> bool` | True if any ancestor is one of the verbatim-class environments (`verbatim`, `Verbatim`, `Code`, `CodeInput`, `CodeOutput`, `Sinput`, `Soutput`, `Scode`, `Schunk`, `CodeChunk`) or a verbatim-class macro (`\verb`, `\code`). |
| `_is_inside_comment` | `_is_inside_comment(node: Node) -> bool` | True for `LatexCommentNode`. (Trivial; exposed for grep-locality.) |

### Invariants

| # | Invariant | Enforcer |
|---|---|---|
| H-1 | `_walk(None)` and `_walk([])` yield zero nodes (no `StopIteration`-on-first-call trap; just an empty generator). | `tests/unit/journals/jss/rules/test_helpers.py` |
| H-2 | `_walk(nodelist)` yields nodes in pre-order: parent before children. | `test_helpers.py` |
| H-3 | `_macro_args_text(macro, parent, idx)` returns `""` when the macro has no braced-group arg. Never raises. | `test_helpers.py` |
| H-4 | `_is_inside_verbatim(ancestors)` is pure (no side effects) and O(len(ancestors)). | `test_helpers.py` |
| H-5 | All helpers are deterministic: same inputs → same outputs, no `random`, no `time`, no module-level state. | `test_helpers.py` (spot-check) + Constitution §I |

### Test coverage

`tests/unit/journals/jss/rules/test_helpers.py` hits 100% branch coverage on `_helpers.py`. Target: small, focused tests per helper — no integration-style workload.

## 4. Per-category rule modules

**Location**: `src/texlint/journals/jss/rules/<category>.py`, one file per category in `catalogue.yaml`'s `categories:` field.

### Canonical module shape

```python
"""<Category title> rules for the JSS journal plugin.

One check callable per rule, bound to a module-level Rule object. Rule
metadata comes from _catalogue_data.RULES (generated from catalogue.yaml);
this module never duplicates catalogue text.

Rules in this module (rollout position N of 15):
  - JSS-<CAT>-001 — <one-line description from catalogue>
  - JSS-<CAT>-002 — ...
"""
from __future__ import annotations

from collections.abc import Iterator

from pylatexenc.latexwalker import (
    LatexCharsNode,
    LatexEnvironmentNode,
    LatexGroupNode,
    LatexMacroNode,
    LatexMathNode,
)

from texlint.api import ParsedDocument, Rule, ToolConfig, Violation
from texlint.journals.jss import _catalogue_data
from texlint.journals.jss.rules import _helpers


# ---------------------------------------------------------------------------
# Check callables — one per rule. Name: check_<lowercased_rule_id_underscored>
# ---------------------------------------------------------------------------

def check_jss_<cat>_001(
    doc: ParsedDocument, cfg: ToolConfig
) -> Iterator[Violation]:
    """<one-line from catalogue description>."""
    meta = _catalogue_data.RULES["JSS-<CAT>-001"]
    for tex in doc.tex_files:
        for node in _helpers._walk(tex.nodes):
            if not _matches_pattern(node):
                continue
            line, col = _helpers._lineno_col(tex, node.pos)
            yield Violation(
                file=tex.path,
                line=line,
                column=col,
                rule_id="JSS-<CAT>-001",
                severity=meta["severity"],
                message=meta["message_template"],
                suggestion=_build_suggestion(node),
                fix=None,
            )


# ---------------------------------------------------------------------------
# Rule objects — one per check callable. Keep this block trivial and grep-able.
# ---------------------------------------------------------------------------

def _rule(rule_id: str, check_fn) -> Rule:
    """Construct a Rule from _catalogue_data.RULES metadata."""
    meta = _catalogue_data.RULES[rule_id]
    return Rule(
        id=rule_id,
        category=meta["category"],
        severity=meta["severity"],
        message_template=meta["message_template"],
        authority=meta["authority"],
        check=check_fn,
        formats=None,
    )


jss_<cat>_001 = _rule("JSS-<CAT>-001", check_jss_<cat>_001)
# jss_<cat>_002 = _rule("JSS-<CAT>-002", check_jss_<cat>_002)
# ... one line per rule ...


# ---------------------------------------------------------------------------
# Module-level rules tuple — consumed by JSSJournal.categories()
# ---------------------------------------------------------------------------

rules: tuple[Rule, ...] = (
    jss_<cat>_001,
    # jss_<cat>_002,
    # ...
)
```

## 5. `JSSJournal.categories()` wiring

**Location**: `src/texlint/journals/jss/__init__.py`.

### Shape

```python
def categories(self) -> tuple[RuleCategory, ...]:
    """Assemble the active JSS rule set.

    Lazy imports: each rule module is loaded only when the engine actually
    runs this journal. Third-party consumers that import `texlint` without
    invoking JSS pay nothing for rule-module code or catalogue data.

    Returns categories in ROLLOUT_ORDER (spec 004 FR-007).
    """
    from texlint.api import RuleCategory
    from texlint.journals.jss import _catalogue_data
    from texlint.journals.jss.rules import (
        abbreviations, bibtex, capitalization, citations,
        code_style, code_width, crossrefs, house_style,
        markup, naming, operators, preamble,
        references, structure, typography,
    )

    modules = {
        "preamble": preamble,
        "structure": structure,
        "markup": markup,
        "citations": citations,
        "references": references,
        "bibtex": bibtex,
        "naming": naming,
        "capitalization": capitalization,
        "typography": typography,
        "abbreviations": abbreviations,
        "code_style": code_style,
        "code_width": code_width,
        "operators": operators,
        "crossrefs": crossrefs,
        "house_style": house_style,
    }
    title_map = {
        "preamble": "Preamble",
        "structure": "Structure",
        "markup": "Markup",
        "citations": "Citations",
        "references": "References",
        "bibtex": "BibTeX",
        "naming": "Naming",
        "capitalization": "Capitalization",
        "typography": "Typography",
        "abbreviations": "Abbreviations",
        "code_style": "Code style",
        "code_width": "Code width",
        "operators": "Operators",
        "crossrefs": "Cross-references",
        "house_style": "House style",
    }

    return tuple(
        RuleCategory(id=cat, title=title_map[cat], rules=modules[cat].rules)
        for cat in _catalogue_data.ROLLOUT_ORDER
    )
```

| J-2 | Returned tuple length equals `len(_catalogue_data.ROLLOUT_ORDER)` (=15). | `test_catalogue_registration.py` |
| J-3 | Returned order matches `ROLLOUT_ORDER`. | `test_catalogue_registration.py` |

| J-5 | Every `Rule.id` across all returned categories appears in `_catalogue_data.RULES`. | `test_catalogue_registration.py` (FR-004 bidirectional check) |

## 7. `tools/generate_catalogue_data.py` — codegen script

**Location**: `tools/generate_catalogue_data.py`.
**Invocation**: `python -m tools.generate_catalogue_data`.

### CLI

```
usage: python -m tools.generate_catalogue_data [--check] [--yaml-path PATH] [--output-path PATH]

Options:
  --check       Do not write; exit 0 if the committed _catalogue_data.py
                matches what would be generated, 1 if it drifts.
  --yaml-path   Override the catalogue source path (default:
                specs/003-jss-rule-catalogue/catalogue.yaml).
  --output-path Override the output file path (default:
                src/texlint/journals/jss/_catalogue_data.py).
```

### Behaviour

- Loads `catalogue.yaml` via `yaml.safe_load` (PyYAML dev extra).
- Calls `tools._catalogue_validate.validate(doc)` — refuses to generate if the catalogue is invalid.
- Emits the generated file atomically (tempfile + `os.replace`), matching `render_catalogue.py`'s write pattern.
- `--check` mode compares against the committed file byte-for-byte; exit 1 on drift (CI hook).

### Deterministic output

Same input → same output. Sort order is `(rollout_order_index, rule_id_counter)`. String literals use `repr()` with `ascii_only=True` semantics (backslashes double-escaped, non-ASCII characters as `\uXXXX`).

## 8. No new storage, no new external interfaces

Spec 004 introduces zero new CLI subcommands, zero new SQLite tables, zero new file formats beyond the TOML skip-list (which is spec 002's file extended, not a new one). All new surface area is Python code under `src/texlint/journals/jss/`.

This deliberately matches Constitution §X "small surface" and spec 004 FR-003 "no new runtime dependencies beyond the Python standard library and the existing pylatexenc / bibtexparser / click / rich / jinja2 stack".
