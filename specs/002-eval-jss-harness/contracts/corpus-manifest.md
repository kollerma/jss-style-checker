# Contract ‚Äî Corpus Manifest (Phase B)

Two CSV files. The manifest is authoritative and hand-maintained; the gaps file is generated by `eval-jss corpus fetch`.

## `eval/corpus-manifest.csv`

Authoritative definition of "the corpus" at this git commit. Checked in.

### Schema

| Column          | Type     | Required | Description                                                                             |
|-----------------|----------|----------|-----------------------------------------------------------------------------------------|
| `jss_doi`       | string   | no       | JSS DOI if known ‚Äî e.g. `10.18637/jss.v085.i01`. Empty string when the paper has no DOI (arXiv-only, manual). |
| `source`        | enum     | yes      | One of `cran`, `bioc`, `arxiv`, `jss_archive`, `manual`, `swh`, `github`.               |
| `source_id`     | string   | yes      | Package name (`MASS`), arXiv identifier (`2401.12345`), JSS article id (`v085i01`), a `swh:` prefix for SWHIDs, or an `owner/repo` slug for `github`. |
| `version`       | string   | yes      | CRAN `cran_version` (e.g. `7.3-60`), Bioc release (e.g. `3.18`), arXiv `vN` (e.g. `v2`), SWH content id (`sha1_git:...`), the literal `manual`, or ‚Äî for `github` ‚Äî `<commit_sha>:<in-repo-manuscript-path>` (e.g. `c075a0d‚Ä¶:paper/main.tex`). |
| `vignette_file` | string   | yes      | Path *inside* `local_path` to the `.tex` file. E.g. `vignettes/paper.tex`.              |
| `local_path`    | path     | yes      | Filesystem path, relative to the `--target` directory (default `examples/`). Typically `<source>_<source_id>_<version>/`. |
| `sha256`        | hex(64)  | yes      | SHA256 of the fetched tarball or single file, lowercase. Empty for `manual` rows until the file is stable. |

### Encoding

UTF-8, LF newlines, comma-separated, double-quoted where values contain commas. Header row is literal:

```
jss_doi,source,source_id,version,vignette_file,local_path,sha256
```

### Distribution URL derivation

| `source`      | URL                                                                                   |
|---------------|---------------------------------------------------------------------------------------|
| `cran`        | `https://cran.r-project.org/src/contrib/Archive/{source_id}/{source_id}_{version}.tar.gz` |
| `bioc`        | `https://bioconductor.org/packages/{version}/bioc/src/contrib/{source_id}_{bioc_version}.tar.gz` (where `bioc_version` is looked up in the Bioc tarball index for `version`) |
| `arxiv`       | `http://arxiv.org/e-print/{source_id}{version}`                                       |
| `jss_archive` | (no fetch ‚Äî manual placement required; `corpus fetch` logs a gap)                     |
| `manual`      | (no fetch ‚Äî file must already be at `local_path`)                                     |
| `swh`         | `https://archive.softwareheritage.org/api/1/content/{version}/raw/` (where `version` is the full `sha1_git:...` identifier) |
| `github`      | `https://codeload.github.com/{source_id}/tar.gz/{commit_sha}` (the commit SHA is the part of `version` before the first `:`). The fetcher copies the pinned manuscript plus the `.bib` files beside it into `<local_path>/vignettes/`, so `vignette_file` is `vignettes/<basename>`. |

All URLs are **immutable, version-pinned** (Constitution ¬ßXII). Mutable discovery sources (GitHub, arXiv listing API, Zenodo search) are **never** used for fetching ‚Äî they are how the manifest gets populated by hand, not how reviewers reproduce the corpus.

### Example rows

```
jss_doi,source,source_id,version,vignette_file,local_path,sha256
10.18637/jss.v067.i01,cran,lme4,1.1-35.1,vignettes/lmer.tex,cran_lme4_1.1-35.1/,7d8c5a3b...
10.18637/jss.v085.i01,cran,limma,3.58.1,vignettes/intro.tex,bioc_limma_3.58.1/,91abf412...
,arxiv,2401.12345,v2,paper.tex,arxiv_2401.12345_v2/,cc9d0e77...
10.18637/jss.v092.i04,manual,jss-v092-i04,manual,paper.tex,manual_jss-v092-i04/,3b44f9aa...
```

### Validation (enforced at `corpus fetch` start)

1. Header row present and exact.
2. Every data row has exactly 7 columns (accounting for CSV quoting).
3. `source` is one of the enum values.
4. `sha256` is 64 hex chars or empty (only empty allowed for `manual` rows that haven't been frozen yet).
5. `local_path` is relative (no leading `/` or `~`).
6. `local_path` is unique across rows.

Any violation ‚Üí exit 2 with an explicit row-number error.

## `eval/corpus-manifest-gaps.csv`

Generated by `eval-jss corpus fetch` each run. Checked in so reviewers can see what was missing at the commit of record. Truncated and rewritten per invocation.

### Schema

| Column            | Description                                                         |
|-------------------|---------------------------------------------------------------------|
| `manifest_row`    | 1-based index of the originating manifest row (excluding header).    |
| `source`          | Copied from the manifest row.                                       |
| `source_id`       | Copied from the manifest row.                                       |
| `version`         | Copied from the manifest row.                                       |
| `reason`          | Human-readable explanation. One of: `http <status>`, `hash mismatch: expected <a>, got <b>`, `timeout`, `tarslip`, `manual placement required`, `unreadable archive: <detail>`. |
| `ts`              | ISO 8601 UTC when the gap was logged.                               |

### Example

```
manifest_row,source,source_id,version,reason,ts
7,cran,archived-pkg,0.9-2,http 404,2026-04-23T14:30:22Z
12,jss_archive,v092i04,manual,manual placement required,2026-04-23T14:30:25Z
```

### Exit-code implication

`corpus fetch` exits 1 iff the gaps file is non-empty at the end of the run. Exit 2 is reserved for manifest-parse failures.
