# Data Model: `jss-lint init`

**Phase**: 1
**Spec**: [spec.md](spec.md)
**Plan**: [plan.md](plan.md)

## 1. Triage classification

```python
@dataclass(frozen=True)
class RuleTriage:
    rule_id: str
    violation_count: int
    unique_line_count: int
    precision: float | None              # None if rule has no DB row
    classification: Literal["must-fix", "suppressed", "untriaged"]
```

| Classification | When                                                                          |
| -------------- | ----------------------------------------------------------------------------- |
| `must-fix`     | Rule has a DB row AND `precision >= threshold` AND violations exist.           |
| `suppressed`   | Rule has a DB row AND `precision < threshold` AND violations exist.            |
| `untriaged`    | Rule has no DB row AND violations exist (the DB exists but lacks this rule).   |

When the DB is entirely missing, every rule with violations is
classified as `untriaged`.

## 2. Generated TOML shape

```toml
journal = "jss"

# auto-generated by `jss-lint init` on 2026-05-03
# threshold = 0.90 (matches Constitution §VI)

ignore_rules = [
  # JSS-MARKUP-005: precision 0.74 (corpus-wide), below 0.90
  "JSS-MARKUP-005",
  # JSS-NAME-007: precision 0.82, below 0.90
  "JSS-NAME-007",
]
```

Ordering:
- Top-level keys: `journal`, `ignore_rules` (severity overrides
  reserved but empty in this spec — research §8).
- Inside `ignore_rules`: rules sorted by `rule_id` ascending. Each
  preceded by exactly one comment line.

## 3. `init.run` signature

```python
def run(
    path: Path,
    *,
    force: bool = False,
    dry_run: bool = False,
    threshold: float = 0.90,
    db_path: Path | None = None,        # default: eval/precision-history.db
    stdout: TextIO | None = None,
    stderr: TextIO | None = None,
) -> int: ...
```

Returns the CLI exit code. Pure I/O at the boundaries (filesystem
read/write, DB read, stdout/stderr); no global state.

## 4. CLI surface

```text
Usage: jss-lint init [OPTIONS] [PATH]

Options:
  --force                   Overwrite an existing .jss-lint.toml.
  --dry-run                 Print proposed config to stdout; do not write.
  --threshold FLOAT         Precision threshold (default 0.90).
  -h, --help                Show this message and exit.
```

PATH defaults to `.` (the current directory).

## 5. Exit codes

| Outcome                                                              | Code |
| -------------------------------------------------------------------- | ---- |
| Wrote `.jss-lint.toml` successfully                                  | 0    |
| `--dry-run` succeeded                                                | 0    |
| Existing config and no `--force`                                     | 2    |
| `--threshold` out of range                                           | 2    |
| PATH does not exist or is unreadable                                 | 2    |
| Internal lint pass crashed (Constitution §III makes this rare)       | 2    |

## 6. Summary table fields

| Column        | Source                                         |
| ------------- | ---------------------------------------------- |
| Rule          | `RuleTriage.rule_id`                           |
| Violations    | `RuleTriage.violation_count`                   |
| Lines         | `RuleTriage.unique_line_count`                 |
| Precision     | `RuleTriage.precision` (or `n/a` for untriaged)|
| Classification| `RuleTriage.classification`                    |

The table header above the rows shows the totals (across all
rules) for total violations and unique-line violations
(Clarifications §4).
